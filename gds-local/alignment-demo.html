About this demo

This is an interactive demonstration of how the Local Government Architecture Model (LGAM) can be used as a shared framework for understanding a council's technology landscape.

What it shows

The LGAM describes local government technology across 9 layers -from the channels citizens use to reach a council, through the capabilities and business systems that deliver services, down to the infrastructure, security, and data platforms that underpin everything.

This demo maps archetypal systems (the kinds of technology a typical council operates) against those layers, and uses the LGAM's relationship model to derive how they connect. It surfaces:

  • Layer position -where each system sits in the overall architecture
  • Derived dependencies -which systems depend on which, based on the LGAM's domain-level relationship graph
  • Coverage gaps -which areas of the model have no system mapped against them
  • Capability suggestions -what capabilities the LGAM suggests are typically needed for systems in that layer

How it relates to the LGAM

The LGAM is a reference architecture -a shared vocabulary and structure. This demo explores what it looks like to align a council's actual technology landscape to that reference. The taxonomy (layers, domains, capabilities) comes directly from the LGAM. The relationships between systems are derived from the LGAM's own dependency graph, not hand-authored.

The reference data uses generic archetypes ("Social Care System", "IAM Platform") rather than specific vendor products. A council using this approach would map their own specific systems into the same structure.

What this is not

  • This is not a procurement tool or product directory
  • This is not a maturity model or scoring framework
  • This is not a replacement for enterprise architecture tooling
  • This is not an official recommendation -it is an exploration of what alignment to the LGAM could look like in practice

Technical notes

This runs entirely in the browser with no backend. The data model is a JSON structure that could be exported and shared. The relationship derivation uses domain-level dependencies from the LGAM graph model. The demo is built with GOV.UK Frontend and vanilla JavaScript.
