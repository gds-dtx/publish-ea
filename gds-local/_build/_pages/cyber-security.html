    Cyber Security

    About this domain

    Protecting council systems, data, and citizens from threats — and meeting regulatory compliance obligations. Covers network security, endpoint protection, security operations, information governance and compliance, vulnerability management, and supply chain security.

    Why this matters for councils

    Understanding the local government context for cyber security decisions:

    • High-value targets — councils hold sensitive personal data (social care, housing, benefits) and often have legacy infrastructure, making them attractive targets for cyber attacks.
    • Real-world impact — recent attacks on councils (Hackney 2020, Redcar & Cleveland 2020, Gloucester 2021, multiple others) caused months of disruption and cost millions to recover from.
    • Cyber Essentials Plus — councils must meet Cyber Essentials Plus certification, which is required for many central government data flows and grants.
    • GDPR/DPA exposure — data breaches carry significant fines and reputational damage. Councils process large volumes of sensitive personal data across many services.
    • Supply chain risk — councils rely on numerous vendors who access council systems and data. A compromise in the supply chain can propagate rapidly.
    • Resource constraints — smaller councils often lack dedicated security staff. They rely on managed Security Operations Centre (SOC) services or share security functions with neighbouring authorities.
    • Expanded attack surface — the shift to cloud and remote working has significantly expanded the attack surface beyond the traditional network perimeter.
    • Post-PSN security — PSN compliance is being replaced but the security expectations remain. Councils must adopt zero-trust models and secure internet-first connectivity.

    Key capabilities

    The core cyber security capabilities that councils need to protect their technology estate and data.

    Network Security

    Firewalls, intrusion detection/prevention (Intrusion Detection System (IDS)/Intrusion Prevention System (IPS)), Distributed Denial of Service (DDoS) protection, network segmentation, secure internet gateways, and Domain Name System (DNS) filtering. The perimeter (and increasingly internal) protection layer.

    Endpoint Protection

    Anti-malware, Endpoint Detection and Response (EDR), device encryption, patch management. Protects the thousands of council devices — laptops, desktops, mobile devices.

    Security Operations

    Threat detection, Security Information and Event Management (SIEM), incident response, security monitoring. Whether in-house, via managed SOC, or shared with partner authorities. The eyes on the screen 24/7.

    Information Governance & Compliance

    GDPR/DPA compliance, records management, data classification, retention enforcement, breach notification, Data Protection Impact Assessment (DPIA) management. The policies and processes that ensure data is handled correctly.

    Vulnerability Management

    Scanning, penetration testing, security assessments, remediation tracking, Cyber Essentials Plus certification, and Cyber Assessment Framework (CAF) alignment.

    Supply Chain Security

    Vendor security assurance, contract security requirements, Software as a Service (SaaS) risk assessment, third-party access controls, and due diligence on the council's technology supply chain.

    What to consider

    Key questions when assessing cyber security capabilities:

    • Do you have Cyber Essentials Plus certification — and can you maintain it year on year?
    • How quickly can you detect and respond to a security incident — do you have 24/7 monitoring?
    • Do you have visibility of all your internet-facing assets, and are they being actively scanned for vulnerabilities?
    • How do you assess the security posture of your vendors and SaaS providers before granting access?
    • Is your incident response plan tested regularly — when did you last run a tabletop exercise?
    • Can you recover from a ransomware attack — how confident are you in your backup integrity and DR capability?
    • How do you manage patching across your estate — what is your mean time to patch critical vulnerabilities?
    • Do you have data classification in place — do staff know how to handle sensitive information?

    Available platforms & tools

    GDS / Public Sector

    DNS Check — free monitoring service for UK public sector domain security. Alerts councils to DNS configuration issues and potential compromise.

    Vulnerability Monitoring Service — Government Digital Service (GDS) service identifying vulnerabilities in internet-facing services. Available to all public sector organisations at no cost.

    SIEM Integration Service — receive DNS Check and vulnerability data directly into your SIEM for correlation with other security events.

    NCSC Active Cyber Defence — suite of free National Cyber Security Centre (NCSC) services (Mail Check, Web Check, Early Warning) available to all UK public sector organisations.

    Cyber Essentials — NCSC-backed certification scheme that provides a baseline of cyber security controls for organisations.

    Commercial (commonly used by councils)

    • Microsoft Defender for Endpoint / Microsoft Sentinel — EDR and SIEM integrated with M365, the natural choice for Microsoft-ecosystem councils.
    • CrowdStrike Falcon — endpoint detection and response platform used by councils requiring best-of-breed EDR.
    • Sophos / Trend Micro — endpoint protection and firewall solutions common in smaller councils.
    • Splunk / LogRhythm / Rapid7 — SIEM platforms for security event correlation and threat detection.
    • Qualys / Tenable / Rapid7 InsightVM — vulnerability scanning and management platforms.
    • Managed SOC providers — e2e-assure, Bridewell, NCC Group, Socura offer outsourced security operations for councils without in-house capability.

    Standards & guidance

    Relevant standards and guidance for cyber security decisions:

    • Government Security — home of security strategies, standards, and policies for UK government
    • Secure by Design — guidance on cyber security practices for digital services
    • NCSC guidance — National Cyber Security Centre guidance library
    • Minimum Cyber Security Standard — baseline security expectations for government
    • Securing Government Email — email security guidance (Domain-based Message Authentication, Reporting and Conformance (DMARC), Sender Policy Framework (SPF), DomainKeys Identified Mail (DKIM))
    • Cyber Assessment Framework (CAF) — NCSC framework for organisations managing cyber risks

    Commercial frameworks

    Crown Commercial Service (CCS) agreements relevant to cyber security procurement:

    • Cyber Security Services — CCS agreement for managed security services, SOC, incident response
    • Security Physical, Technical and Support Services — CCS agreement for security products and services
    • Network Services — CCS agreement including network security and DNS services

    Connections to service delivery

    Cyber Security protects the entire LGAM service delivery stack:

    • All IT Systems — endpoint protection, vulnerability scanning, and patch management cover every council system
    • Council Interfaces (Website, Apps, API Gateway) — network security and Web Application Firewall (WAF) protect public-facing services
    • Platform & Infrastructure — security operations monitor the hosting and network layers
    • Integration — supply chain security assesses the vendors whose APIs and platforms councils connect to
    • Identity & Access Management (in Platform layer) — works alongside security operations to detect compromised accounts
