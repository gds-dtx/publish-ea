    Architecture view: Adult Social Care

    About this view

    An interactive graph showing a council's adult social care technology architecture, using LGAM layers and relationship types.

    Nodes represent specific systems, services, and components. Edges represent LGAM relationships (ROUTES TO, USES, SUPPORTS, ENABLES, SECURES). Colours indicate which LGAM layer the element belongs to.

    This is illustrative — each council would populate with their own systems. A mapping to ArchiMate terminology is provided below for architects who need formal modelling interoperability.

    ArchiMate translation

    For architects who use ArchiMate, the following mapping shows how to translate LGAM concepts into ArchiMate terminology. Note that LGAM is its own framework — ArchiMate does not map perfectly onto it, particularly for the upper layers where LGAM models a user journey (left-to-right flow) while ArchiMate models service provision (bottom-to-top).

    Element type mapping

    LGAM Layer ArchiMate Element Types Notes
    Public ChannelsBusiness InterfaceHow citizens reach services
    Council InterfacesApplication InterfaceSystems handling citizen interactions
    CapabilitiesApplication Component / ServiceReusable application services
    Business AreasBusiness ProcessDomain-specific functions
    Corporate AreasBusiness FunctionBack-office functions
    Foundational TechnologyNode, Technology ServiceInfrastructure platforms
    IntegrationApplication Component, PathArchiMate treats integration as paths/networks, not elements
    SecurityConstraint, Technology ServiceArchiMate treats security as cross-cutting constraints
    DataData Object, ArtifactData assets and platforms

    Relationship mapping

    LGAM Relationship Nearest ArchiMate Equivalent Fit
    ROUTES TOTriggeringPartial — ArchiMate has no direct "routes" concept
    USESServing (reversed direction)Partial — ArchiMate "Serving" flows from provider to consumer
    SUPPORTSServingGood fit
    ENABLESRealizationGood fit for FT→Capabilities
    SECURESInfluence / AssociationPoor — ArchiMate doesn't model "secures" as a relationship
    CONNECTSFlowGood fit

    Using LGAM in ArchiMate tools

    To model your council's architecture in Archi (free, open source) aligned to LGAM:

    1. Create a grouping element for each LGAM layer
    2. Add your council's specific systems as elements within each group, using the ArchiMate types above
    3. Draw relationships between elements — use ArchiMate's Serving/Realization/Flow where they fit; use Association for LGAM relationships that don't map cleanly
    4. Use ArchiMate viewpoints to create focused views (e.g., Application Cooperation for integration, Layered for the full stack)
