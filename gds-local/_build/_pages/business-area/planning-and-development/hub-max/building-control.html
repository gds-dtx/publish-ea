      Building Control

      The technical architecture supporting the enforcement of building regulations, structural safety, and on-site compliance inspections.

      Unlike Development Control, Building Control is rarely public-facing. It is a highly technical, field-centric service. Its architecture must prioritize offline-capable mobile applications for site inspectors, the ingestion of complex 3D technical drawings (BIM), and rigid, immutable audit trails to comply with the Building Safety Act and the Building Safety Regulator (BSR).

      The LGAM technology stack

      Delivering this journey requires specific capabilities focused heavily on mobile workforce enablement and secure data storage.

      LGAM Taxonomy Node Domain Specific Implementation
      Mobile working (Field apps) Offline-first tablet applications allowing inspectors to log site evidence (photos, notes) in basements or rural areas without 4G/5G, syncing to the back-office upon reconnection.
      Document and records management Highly secure, scalable storage capable of handling massive Building Information Modeling (BIM) files and CAD drawings, often subject to commercial confidentiality.
      Geospatial data / Routing Dynamic routing algorithms to optimize the daily schedules of field inspectors across the borough, reducing travel time and fuel consumption.
      Partner (B2B) integration API linkages to the national Building Safety Regulator (BSR) to automatically pass high-risk building (HRB) data upstream.

      The technical lifecycle

      This section outlines the business process at each phase of a building control application. Expand each phase to compare the baseline legacy architecture against modern target states.

      1 Submission & Ingestion

      • Business Process
      • Legacy Anti-patterns
      • Target Architecture

      The process: Builders, architects, or homeowners submit technical structural drawings before work commences. The key architectural challenge here is securely handling massive file sizes (e.g. 500MB CAD files).

      Required LGAM Taxonomy Nodes:

      Forms Document and records management Payments
      Anti-pattern: Disconnected large file transfer

      Because standard council web forms limit file sizes (e.g., to 10MB), applicants email links via Dropbox or WeTransfer for their massive CAD files. Officers manually download these and store them on unmanaged local network drives, breaking the audit trail.

      Best Practice: Unified blob storage and pre-signed URLs

      Modern submission portals utilize cloud-native object storage (e.g., AWS S3, Azure Blob). When a user needs to upload a massive BIM model, the frontend requests a temporary, pre-signed URL from the API Gateway, allowing the user's browser to stream the file directly into the council's secure cloud storage, bypassing the web server's memory limits.

      Real-world impact
      • Information Governance

        Eliminates "Shadow IT" (staff using personal Dropbox accounts). Highly sensitive structural drawings (e.g., for banks or schools) are stored in encrypted, auditable enterprise environments.

      • System Performance

        Prevents massive file uploads from crashing the primary web forms server, ensuring high availability for other citizens trying to access council services.

      2 Plan Checking & Assessment

      • Business Process
      • Legacy Anti-patterns
      • Target Architecture

      The process: Building Control surveyors must thoroughly review complex structural plans and calculations against national Building Regulations (e.g. fire safety, thermal performance, escape routes) before any construction work is permitted to begin.

      Required LGAM Taxonomy Nodes:

      Document and records management Workflow Business rule engines
      Anti-pattern: Flattening 3D BIM to 2D PDFs

      Architects design modern buildings using rich, data-heavy 3D Building Information Models (BIM). However, legacy council systems force applicants to "flatten" these models into hundreds of static 2D PDF blueprints for submission. Surveyors then spend hours manually measuring distances on screen, losing all the embedded structural metadata.

      Best Practice: Automated BIM Interrogation

      Modern architectures accept native BIM formats (such as IFC). Instead of manual measurement, the compliance engine automatically runs rule-sets against the model's spatial metadata—for example, instantly calculating if the distance between any desk and a fire exit exceeds the maximum allowed by regulations, flagging only the failures for the human surveyor to review.

      Real-world impact
      • Regulatory Accuracy

        Completely removes human error from complex geometric calculations, ensuring higher building safety standards.

      • Accelerated Approvals

        Reduces the plan checking time from weeks down to days by automating the bulk of the compliance math, allowing highly-trained surveyors to focus purely on exceptions.

      3 Site Inspections & Fieldwork

      • Business Process
      • Legacy Anti-patterns
      • Target Architecture

      The process: The core of Building Control. Surveyors travel to active construction sites to inspect foundations, damp proofing, and steelwork. They must record evidence (photos, notes, pass/fail status) and link it precisely to the property record.

      Required LGAM Taxonomy Nodes:

      Mobile working Geospatial data
      Anti-pattern: "Online-only" web apps in the field

      Deploying standard council web-portals to tablets. When inspectors go into concrete basements or rural development sites without 4G, the application drops the session, forcing them to use pen-and-paper and transcribe notes back at the office.

      Best Practice: Offline-first Mobile Architecture

      Target architectures utilize native or PWA (Progressive Web App) mobile technologies designed as "offline-first". Inspectors download their daily case lists and mapping tiles to a local encrypted SQLite database on the device via Wi-Fi in the morning. They log data seamlessly without signal; the app's sync-engine automatically handles the API payload delivery to the back-office once 4G is restored.

      Real-world impact
      • Operational Efficiency

        Recovers up to an hour per inspector, per day, by eliminating the need to return to the office to manually upload photographs from a digital camera and type up paper notes.

      • Data Quality & Compliance

        Photographic evidence is automatically tagged with GPS coordinates and timestamps at the exact moment of capture, creating an unassailable legal audit trail.

      4 Certification & Compliance

      • Business Process
      • Legacy Anti-patterns
      • Target Architecture

      The process: Following satisfactory inspections, the council issues a formal Completion Certificate. Crucially, post-Grenfell legislation requires stringent, ongoing data sharing regarding High-Risk Buildings (HRBs) with the central Building Safety Regulator (BSR).

      Required LGAM Taxonomy Nodes:

      Partner (B2B) integration Data quality and governance
      Anti-pattern: "Swivel-chair" regulatory reporting

      Building Control officers must manually run reports in their local system, export them to Excel, and manually upload them to a central government portal. This creates a lag in safety data and a high risk of transcription errors regarding critical structural information.

      Best Practice: B2B API integrations and immutable logs

      Target architectures automate statutory compliance. When an inspector signs off a High-Risk Building phase, the workflow engine automatically triggers a webhook. The API Gateway formats the required payload (anchored by the UPRN) and pushes it synchronously to the Building Safety Regulator's APIs.

      Real-world impact
      • Regulatory Compliance

        Ensures absolute, real-time compliance with the Building Safety Act without manual overhead. Central regulators have immediate visibility of high-risk site statuses.

      • Liability Reduction

        Immutable, system-to-system data transfers prevent human error in safety reporting, securing the "golden thread" of building information and reducing the council's legal risk exposure.

      Target data models

      Because Building Control is fundamentally tied to physical structures, the UPRN is non-negotiable. Furthermore, systems must be capable of adhering to the data schemas mandated by the Building Safety Act.

      Data Model / Standard Architectural Purpose
      The Golden Thread (UPRN) Every inspection, enforcement notice, and drawing must be fundamentally anchored to a UPRN, ensuring a complete safety history of a property that survives changes in ownership or system migrations.
      Building Information Modeling (BIM) Standardized 3D object-oriented models containing exact structural data. Future target states require storage solutions capable of ingesting IFC (Industry Foundation Classes) file formats, rather than just flat 2D PDFs.

      Platforms & tools

      The market for Building Control software is heavily focused on integrated enterprise platforms that bundle back-office workflow with mobile inspection capabilities.

      • NEC Assure: Widely used cloud platform covering Building Control workflow and mobile field working.
      • Idox Uniform / OnSite: Established enterprise suite offering deep back-office capability paired with dedicated tablet applications for offline fieldwork.
      • Tascomi (Idox): Cloud-native regulatory services software with responsive web interfaces often utilized by mobile officers.
      • Agile Applications: Specialized local government platform covering regulatory services.
